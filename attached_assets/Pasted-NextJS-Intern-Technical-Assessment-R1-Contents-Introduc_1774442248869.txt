Introduction
Create a SocialConnect Web Application, a social media application using NextJS. SocialConnect allows users to share posts, connect with others, and discover content through a personalized feed experience.

Core Functionality
Users can register, create profiles, post content with images, like and comment on posts, and discover content through a personalised feed.
Key Features
Authentication System: JWT-based authentication with login/register/logout
User Profiles: Basic profiles with bio, avatar, follower/following counts
Content Creation: Text posts with single image upload capability
Social Interactions: Like posts, basic comment system
Personalised Feed: Chronological feed of public posts

API Endpoints Overview
Authentication System
JWT-based authentication

User Registration
Endpoint: POST /api/auth/register/
Fields: email, username, password, first_name, last_name
Username must be unique and follow validation rules (3-30 chars, alphanumeric + underscore)
User Login
Endpoint: POST /api/auth/login/
Accept email OR username with password
Return access token and user profile data
Track last login timestamp
Logout
Endpoint: POST /api/auth/logout/
User Management
User Profile System
Basic user profiles

Profile CRUD Operations
Get Profile: GET /api/users/{user_id}/
Update Own Profile: PUT/PATCH /api/users/me/
Profile fields: bio (max 160 chars), avatar_url, website, location
Avatar URL stored as a string (see Optionals for file upload)
User Statistics
Include in profile response: posts_count
Basic user listing: GET /api/users/
Posts and Content
Post Management System
Simple content creation with single image support

Post CRUD Operations
Create Post: POST /api/posts/
Get Post: GET /api/posts/{post_id}/
Update Own Post: PUT/PATCH /api/posts/{post_id}/
Delete Own Post: DELETE /api/posts/{post_id}/
List All Posts: GET /api/posts/ (with pagination)
Post Content Features
Text content with character limit (280 chars)
Single image upload
Post Model Requirements
Copy
# Required fields for Post model
- content (TextField, max_length=280)
- author (ForeignKey to User)
- created_at, updated_at (DateTimeField)
- image_url (URLField, optional)
- is_active (BooleanField, default=True)
Image Handling
Support JPEG, PNG formats only
Maximum file size: 2MB per image
Upload to Supabase Storage and store URL in database
Basic image validation (file type and size)
Social Features
Likes & Comments
Like Operations
Like Post: POST /api/posts/{post_id}/like/
Unlike Post: DELETE /api/posts/{post_id}/like/
Comment Operations
Add Comment: POST /api/posts/{post_id}/comments/
List Comments: GET /api/posts/{post_id}/comments/
Delete Own Comment: DELETE /api/posts/{post_id}/comments/{comment_id}/
Personalised Feed
Endpoint: GET /api/feed/
Returns a chronological list of public posts
Follow/Unfollow System
Follow User: POST /api/users/{user_id}/follow/
Unfollow User: DELETE /api/users/{user_id}/follow/
Get User Followers: GET /api/users/{user_id}/followers/
Get User Following: GET /api/users/{user_id}/following/
Follow Model: follower, following (ForeignKey to User), created_at; unique together constraint
When implemented, the personalised feed should show posts from followed users
Avatar Upload to Supabase Storage
Allow users to upload a profile picture directly to Supabase Storage
Store the resulting URL in the user profile
Basic file validation (JPEG/PNG, max 2MB)
Denormalized like_count and comment_count on Post model
Add like_count and comment_count as PositiveIntegerField (default=0) on the Post model
Keep these fields in sync when likes/comments are added or removed
Technology Stack
Backend Framework: NextJS
Database: PostgreSQL (via Supabase)
Authentication: Using any JWT package or Supabase Authentication
File Storage: Supabase Storage for images and media
Frontend: Next.js with TypeScript
UI Framework: Tailwind CSS with shadcn/ui components
Deployment: Vercel/Netlify
